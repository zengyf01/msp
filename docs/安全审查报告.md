## 二、数据传输边界 - 防止数据出域

### 2.1 gRPC 通信未使用 TLS
- **文件**: `msp-node/src/msp_node/protos/msp_node.proto:34`
- **问题**: `service MSPNode` 定义无任何 TLS 配置；`grpc_server.py:727` 使用 `add_insecure_port()`
- **风险等级**: 高
- **建议**: 生成 TLS 证书，配置 gRPC TLS 加密连接

### 2.2 Kuscia API 使用 HTTP 协议
- **文件**: `msp-backend/msp-scheduler/src/main/resources/application.yml:29`
- **问题**: `protocol: http`（同时 values.yaml:84 配置为 `tls` 但实际未生效）
- **风险等级**: 高
- **建议**: 统一使用 `https` 并配置有效证书

### 2.3 Kuscia WebClient 跳过证书验证
- **文件**: `msp-backend/msp-kuscia/src/main/java/com/msp/kuscia/config/KusciaWebClientConfig.java:52-55`
- **问题**: `InsecureTrustManagerFactory.INSTANCE` 接受任意证书
- **风险等级**: 高
- **建议**: 配置受信任的 CA 证书或指定证书文件

### 2.4 节点间 HTTP 通信无加密
- **文件**: `msp-node/src/msp_node/runners/dag_executor.py:543,574`
- **问题**: 多方 PSI 密钥交换和加密 ID 提交使用 HTTP POST
- **风险等级**: 高
- **建议**: 使用 HTTPS 或在应用层实现端到端加密

## 四、任务调度安全 - 数据泄露风险

### 4.1 DAG 定义通过 HTTP 传输包含敏感信息
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/RayTaskExecutor.java:261-267`
- **问题**: `requestBody.put("dag_definition", customizedDagDefinition)` 明文传输 DAG 定义，其中可能包含数据源凭证
- **风险等级**: 高
- **建议**: 使用 HTTPS + mTLS，确保只有授权节点能接收任务

### 4.2 数据源密码在任务分发时被传递
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/RayTaskExecutor.java:267`
- **问题**: `requestBody.put("data_sources", request.getInputs())` 将数据源配置（包含明文密码）发送到各节点
- **风险等级**: 高
- **建议**: 仅传递数据源 ID，让各节点从本地安全存储获取凭证

### 4.3 自定义代码执行存在代码注入风险
- **文件**: `msp-node/src/msp_node/runners/custom_code_runner.py:47`
- **问题**: `exec(code, exec_globals)` 直接执行用户提供的代码，无任何沙箱隔离
- **风险等级**: 高
- **建议**:
  - 使用 Docker 容器或虚拟机隔离执行
  - 实现细粒度的函数白名单
  - 禁用危险函数（如 `os`、`subprocess`、`open` 等）

### 4.4 任务执行代码生成存在路径遍历风险
- **文件**: `msp-backend/msp-kuscia/src/main/java/com/msp/kuscia/service/TaskSpecGenerator.java:420-422`
- **问题**: `sb.append("path = ").append(path).append("'")` 用户控制的 `path` 直接拼接进生成的 Python 代码
- **风险等级**: 高
- **建议**: 对 `path` 进行严格校验，只允许字母数字和特定分隔符

## 五、API 安全

### 5.1 CORS 配置过度宽松
- **文件**: `msp-backend/msp-gateway/src/main/resources/application.yml:51`
- **问题**: `allowedOrigins: "http://localhost:3000"` 仅允许本地开发，但 `allowedHeaders: "*"` 和 `allowCredentials: true` 组合存在风险
- **风险等级**: 中
- **建议**: 生产环境明确列出允许的域名

### 5.2 密码使用 SHA-256 简单哈希（无盐）
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/UserService.java:222-230`
- **问题**: `hashPassword` 使用裸 SHA-256，无 salt 无迭代
- **风险等级**: 高
- **建议**: 使用 BCrypt、Argon2 或 PBKDF2 等专业密码哈希算法

### 5.3 缺少请求体大小限制
- **文件**: `msp-backend/msp-gateway/src/main/resources/application.yml`
- **问题**: 无 `body-size` 限制，可能导致 DoS
- **风险等级**: 中
- **建议**: 配置请求体大小限制

### 5.4 节点 API 无认证机制
- **文件**: `msp-node/src/msp_node/transport/grpc_server.py:374-441`
- **问题**: `/api/v1/execute/dag` 和 `/api/v1/psi/*` 端点无任何认证
- **风险等级**: 高
- **建议**: 实现节点间 mutual TLS 或共享密钥认证

## 七、DAG 组件安全审查

### 7.1 PSI 对齐结果直接暴露
- **文件**: `msp-backend/msp-kuscia/src/main/java/com/msp/kuscia/service/TaskSpecGenerator.java:345`
- **问题**: `aligned_df = sf.reveal(result)` 直接暴露对齐结果，违反"数据不可见"原则
- **风险等级**: 高
- **建议**: PSI 结果应在密态下使用，不应明文 reveal

### 7.2 write_csv 结果明文写入文件系统
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/TaskSchedulerImpl.java:940-978`
- **问题**: DAG 执行结果以 CSV 明文写入 `/tmp` 目录，文件权限配置不当可导致泄露
- **风险等级**: 高
- **建议**: 结果应写入隔离的安全存储，限制文件权限为 600

### 7.3 节点间 HTTP 调用无认证
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/RayTaskExecutor.java:440-461`
- **问题**: `String url = "http://" + nodeAddress + "/api/v1/execute/dag"` 无任何认证机制
- **风险等级**: 高
- **建议**: 实现双向认证（mTLS）或使用签名机制验证请求来源

### 7.4 数据源凭证通过环境变量注入
- **文件**: `msp-backend/msp-kuscia/src/main/java/com/msp/kuscia/service/TaskSpecGenerator.java:211-219`
- **问题**: `os.environ.get(f'{prefix}_PASS', ...)` 密码从环境变量获取，日志可能泄露
- **风险等级**: 高
- **建议**: 使用 KMS 或 Vault 管理凭证，避免密码出现在环境变量中

### 7.5 DAG组件兼容性检查存在绕过
- **文件**: `msp-frontend/src/views/DAGDesignerView.vue:380-384`
- **问题**: 未知组件 ID 直接放行 `if (!toDef) return true`，允许任意连接
- **风险等级**: 中

### 7.6 数据源所有权验证存在回退逻辑绕过

- **风险等级**: 中

### 7.7 Kuscia 提交失败后自动回退到 ray 模式
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/TaskSchedulerImpl.java:114-116`

### 7.8 节点地址解析存在硬编码中文名
- **文件**: `msp-backend/msp-scheduler/src/main/java/com/msp/scheduler/service/RayTaskExecutor.java:463-486`

### 7.11 write_table 组件未实现

## 八、机密容器设计对比

### 8.1 CoCo 的"数据可用不可见"实现方式

Confidential Containers 通过以下机制实现隐私保护：

| 机制 | CoCo 实现 | MSP 当前实现 | 差距 |
|------|-----------|--------------|------|
| **TEE 硬件加密** | AMD SEV-SNP/Intel TDX 硬件内存加密 | 无 | MSP 缺少硬件级加密 |
| **远程认证** | RATS 架构，KBS 验证 TEE 证据后才分发密钥 | 无 | MSP 无法验证节点可信度 |
| **镜像加密** | 加密容器镜像，密钥通过认证后获取 | 明文存储 | MSP 数据在节点上明文 |
| **密钥分发** | KEK → LEK 分层加密，JWE格式封装 | 环境变量注入 | MSP 密钥管理不安全 |
| **数据边界** | TEE 硬件边界，数据不出硬件 | 节点间 HTTP 明文通信 | MSP 数据可被网络截获 |

### 8.2 MSP 核心差距分析

```
CoCo 信任模型:
┌─────────────────────────────────────┐
│ Untrusted Cloud Infrastructure │
│  (CSP 无法访问 TEE 内部数据)         │
└─────────────────────────────────────┘
                    │
            Hardware TEE Boundary
                    ▼
┌─────────────────────────────────────┐
│     Trusted Enclave (Guest VM)       │
│  -硬件内存加密                      │
│  - 远程认证后才获取密钥               │
│  - 数据在 TEE 内部才可见              │
└─────────────────────────────────────┘

MSP 当前信任模型:
┌─────────────────────────────────────┐
│     调度器 (Scheduler)                 │
│  - 明文 HTTP 分发任务                 │
│  - 数据源密码传递到节点 │
└─────────────────────────────────────┘
                    │
            网络传输（无 TLS）
                    ▼
┌─────────────────────────────────────┐
│     计算节点 (Node)                   │
│  -节点可直接访问数据源               │
│  - PSI 结果用 sf.reveal 明文暴露 │
│  - 结果写入 /tmp 明文文件 │
└─────────────────────────────────────┘
```

### 8.3 MSP 改进建议（参考 CoCo 设计）

1. **引入 TEE 硬件支持**
   - 评估 AMD SEV-SNP 或 Intel TDX 支持
   -节点在 TEE 内运行计算任务

2. **实现远程认证机制**
   - 参考 CoCo 的 KBS + Attestation Service 架构
   - 节点认证后才可接收任务和数据

3. **加密数据传输**
   - 节点间通信使用 TLS/mTLS
   - DAG 定义加密后传输

4. **密钥分层管理**
   - 参考 KEK → LEK 分层加密方案
   - 使用 KMS 管理根密钥

5. **PSI 结果密态处理**
   - PSI 对齐结果不应明文 reveal
   - 通过安全聚合机制处理结果

### 11.4 高危问题：PSI 实现缺陷

#### 11.4.1 PSI 结果明文暴露

- **文件**: `dag_executor.py:1238-1243`
- **问题**: 树结构特征重要性明文返回
- **风险**: 可利用模型结构逆向训练数据分布
- **建议**: 对模型参数进行差分隐私处理

### 11.6 中危问题

#### 11.6.1 SGB 预测使用伪随机数
- **文件**: `dag_executor.py:1293-1294`
- **问题**: `np.random.rand(len(df))` 生成伪预测概率
- **风险**: 预测结果不是真正基于模型，误导业务决策
- **建议**: 实现真正的 SGB 预测逻辑

#### 11.6.2 ECDH PSI 确定性加密风险
- **文件**: `ecdh_psi.py:113-128`
- **问题**: HMAC-SHA256 确定性加密意味着相同 ID 产生相同密文
- **风险**: 可被统计攻击识别特定 ID
- **建议**: 使用随机化加密或添加盐值

### 11.7 梯度更新和模型逆向风险分析

#### 11.7.1 梯度暴露风险
- **当前问题**: `ss_glm_train` 使用 sklearn 本地训练，梯度直接在本地计算
- **风险**: 如果使用真正的联邦学习，各方梯度更新可能泄露原始数据
- **攻击方式**: 攻击者可通过梯度逆向原始训练样本
- **建议**: 实现安全聚合（Secure Aggregation）协议

#### 11.7.2 模型参数逆向风险
- **当前问题**: `model_params` 包含完整的模型系数、截距、特征名
- **风险**: 线性模型系数可直接反映特征重要性，结合特征工程知识可推断原始数据
- **建议**: 对模型参数进行差分隐私处理

#### 1.1 核心架构组件

| 组件 | 全称 | 功能描述 |
|------|------|----------|
| **SPU** | Secure Processing Unit | 安全处理单元，实现 MPC 协议，支持 SEMI2K、ABY3、REF2K |
| **HEU** | Homomorphic Encryption Unit | 同态加密单元，支持加法/乘法同态加密 |
| **TEEU** | Trusted Execution Environment Unit | 可信执行环境单元，支持 Intel SGX、AMD SEV |
| **PYU** | Python Unit | Python 计算单元，代表各方本地计算 |
| **Link** | - | 节点间安全通信链路 |

#### 1.2 支持的隐私计算协议

| 协议类型 | SecretFlow 支持 | MSP 支持状态 |
|----------|-----------------|--------------|

| **DP** | 差分隐私噪声添加 | **未实现** |
