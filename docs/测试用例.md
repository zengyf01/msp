# 密算平台 (MSP) 测试用例

## 基础信息
- **测试环境**: http://localhost:8090 (Gateway)
- **默认用户**: admin / admin123
- **API前缀**: /api/v1/msp

---

## 一、用户认证模块

### 1.1 登录

**请求**
```bash
POST /api/v1/msp/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

**预期响应**
```json
{
  "success": true,
  "code": "200",
  "message": "success",
  "data": {
    "token": "YWRtaW4tZGVmYXVsdC1pZDoxNzc5MjU4MjU3NzMy",
    "userId": "admin-default-id",
    "username": "admin"
  }
}
```

### 1.2 登出

**请求**
```bash
POST /api/v1/msp/auth/logout
Header: X-TOKEN: <登录返回的token>
```

**预期响应**
```json
{
  "success": true,
  "code": "200",
  "message": "success"
}
```

### 1.3 获取当前用户

**请求**
```bash
GET /api/v1/msp/auth/current
Header: X-TOKEN: <登录返回的token>
```

**预期响应**
```json
{
  "success": true,
  "code": "200",
  "message": "success",
  "data": {
    "userId": "admin-default-id",
    "username": "admin",
    "role": "ADMIN",
    "enabled": true
  }
}
```

### 1.4 创建用户

**请求**
```bash
POST /api/v1/msp/auth/register
Content-Type: application/json
Header: X-TOKEN: <管理员token>

{
  "username": "testuser",
  "password": "test123",
  "role": "USER"
}
```

**预期响应**
```json
{
  "success": true,
  "code": "200",
  "message": "success",
  "data": "<新用户ID>"
}
```

## 二、任务管理模块

### 2.1 创建任务

**请求**
```bash
POST /api/v1/msp/tasks
Content-Type: application/json
Header: X-TOKEN: <token>

{
  "name": "PSI任务测试",
  "type": "PSI",
  "algorithm": "RSA",
  "description": "测试隐私求交任务",
  "participants": ["node1", "node2"],
  "inputs": {
    "dataSourceId": "<数据源ID>",
    "columns": ["id", "name"]
  },
  "parameters": {
    "matchType": "INTERSECTION"
  }
}
```

**预期响应**
```json
{
  "success": true,
  "code": "200",
  "message": "success",
  "data": "<任务ID>"
}
```

**请求**
```bash
GET /api/v1/msp/tasks
Header: X-TOKEN: <token>
```

- `size` (可选, 默认10)

```json
{
  "success": true,
  "code": "200",
  "message": "success",
  "data": {
    "content": [
      {
        "taskId": "<任务ID>",
        "name": "PSI任务测试",
```

## 六、集成测试场景

### 6.1 完整任务创建与执行流程

```
1. 管理员登录
   POST /api/v1/msp/auth/login
   -> 获取token

2. 创建节点
   POST /api/v1/msp/nodes
   -> 获取nodeId

3. 创建数据源
   POST /api/v1/msp/datasources
   -> 获取dataSourceId

4. 创建任务
   POST /api/v1/msp/tasks
   -> 获取taskId

5. 查询任务列表确认
   GET /api/v1/msp/tasks

6. 更新任务状态为RUNNING
   PUT /api/v1/msp/tasks/<taskId>/status
   Body: {"status": "RUNNING"}

7. 查询任务详情
   GET /api/v1/msp/tasks/<taskId>

8. 更新任务状态为COMPLETED
   PUT /api/v1/msp/tasks/<taskId>/status
   Body: {"status": "COMPLETED"}

9. 查询审计日志确认操作记录
   GET /api/v1/msp/audit-logs?userId=<userId>
```

### 6.2 多任务并发状态查询

```
1. 批量创建5个任务
   POST /api/v1/msp/tasks (x5)

2. 批量更新任务状态
   PUT /api/v1/msp/tasks/<taskId>/status (每个任务)

3. 按状态过滤查询
   GET /api/v1/msp/tasks/status/RUNNING
   GET /api/v1/msp/tasks/status/COMPLETED

4. 验证状态一致性
```

## 七、错误码说明

| 错误码 | 说明 |
|--------|------|
| AUTH_FAILED | 认证失败（用户名/密码错误） |
| USER_DISABLED | 用户已被禁用 |
| USER_NOT_FOUND | 用户不存在 |
| TASK_NOT_FOUND | 任务不存在 |
| NODE_NOT_FOUND | 节点不存在 |
| DATA_SOURCE_NOT_FOUND | 数据源不存在 |
| UNAUTHORIZED | 未授权（缺少token） |
| FORBIDDEN | 禁止访问（权限不足） |
| INTERNAL_ERROR | 内部服务器错误 |

---

## 八、测试注意事项

1. **Token传递**: 除登录接口外，所有请求都需要通过 `X-TOKEN` header 传递token
2. **时间戳**: 所有时间字段均为毫秒级Unix时间戳
3. **分页**: 列表接口默认使用分页，页面从0开始计数
4. **数据依赖**: 删除节点前需确保没有关联的数据源
5. **并发安全**: 不建议在同一数据上执行并发更新操作
