## 基础信息
- **测试环境**: http://localhost:8090 (Gateway)
- **默认用户**: admin / admin123
- **API前缀**: /api/v1/msp

## 一、用户认证模块

## 六、集成测试场景

### 6.1 完整任务创建与执行流程

```
1. 管理员登录
   POST /api/v1/msp/auth/login
   -> 获取token

2. 创建节点
   POST /api/v1/msp/nodes
   -> 获取nodeId

3. 创建数据源
   POST /api/v1/msp/datasources
   -> 获取dataSourceId

4. 创建任务
   POST /api/v1/msp/tasks
   -> 获取taskId

5. 查询任务列表确认
   GET /api/v1/msp/tasks

6. 更新任务状态为RUNNING
   PUT /api/v1/msp/tasks/<taskId>/status
   Body: {"status": "RUNNING"}

7. 查询任务详情
   GET /api/v1/msp/tasks/<taskId>

8. 更新任务状态为COMPLETED
   PUT /api/v1/msp/tasks/<taskId>/status
   Body: {"status": "COMPLETED"}

9. 查询审计日志确认操作记录
   GET /api/v1/msp/audit-logs?userId=<userId>
```

### 6.2 多任务并发状态查询

```
1. 批量创建5个任务
   POST /api/v1/msp/tasks (x5)

2. 批量更新任务状态
   PUT /api/v1/msp/tasks/<taskId>/status (每个任务)

3. 按状态过滤查询
   GET /api/v1/msp/tasks/status/RUNNING
   GET /api/v1/msp/tasks/status/COMPLETED

4. 验证状态一致性
```

## 七、错误码说明

| 错误码 | 说明 |
|--------|------|
| AUTH_FAILED | 认证失败（用户名/密码错误） |
| USER_DISABLED | 用户已被禁用 |
| USER_NOT_FOUND | 用户不存在 |
| TASK_NOT_FOUND | 任务不存在 |
| NODE_NOT_FOUND | 节点不存在 |
| DATA_SOURCE_NOT_FOUND | 数据源不存在 |
| UNAUTHORIZED | 未授权（缺少token） |
| FORBIDDEN | 禁止访问（权限不足） |
| INTERNAL_ERROR | 内部服务器错误 |

---

## 八、测试注意事项

1. **Token传递**: 除登录接口外，所有请求都需要通过 `X-TOKEN` header 传递token
2. **时间戳**: 所有时间字段均为毫秒级Unix时间戳
3. **分页**: 列表接口默认使用分页，页面从0开始计数
4. **数据依赖**: 删除节点前需确保没有关联的数据源
5. **并发安全**: 不建议在同一数据上执行并发更新操作

## 九、MPC运算模块

#### TC-MPC-001: 两方加法

**测试目标**: 验证MPC加法运算返回正确结果 (100 + 200 = 300)

**前置条件**: 已启动模拟节点node-a和node-b，SecretFlow >= 1.14.0

**测试步骤**:
1. 准备测试数据
```python
# 节点A数据
data_a = [100, 200, 300]

# 节点B数据
data_b = [50, 100, 150]
```

2. 执行MPC加法
```bash
POST /api/v1/msp/tasks
Content-Type: application/json
Header: X-TOKEN: <token>

{
  "name": "MPC加法测试",
  "type": "MPC",
  "algorithm": "semi2k",
  "participants": ["node-a", "node-b"],
  "parameters": {
    "mpc_type": "addition"
  },
  "inputs": {
    "data_a": [100, 200, 300],
    "data_b": [50, 100, 150]
  }
}
```

3. 验证结果
```bash
GET /api/v1/msp/tasks/<taskId>/result
Header: X-TOKEN: <token>
```

**预期结果**:
```json
{
  "status": "ok",
  "mpc_type": "addition",
  "result": [150, 300, 450]
}
```

**执行结果**: ✅ 通过 (容器内测试)
