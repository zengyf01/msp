## 一、项目概述

### 1.1 产品定位

MSP（Multi-party Computation Platform）隐私计算平台，旨在为可信数据空间提供隐私计算能力，解决"数据可用不可见、数据不出域"的核心问题。

### 1.2 核心价值

- **数据安全**：原始数据不出域，只交换计算结果
- **多方协同**：支持多参与方联合计算
- **隐私保护**：基于密码学的隐私保护方案
- **灵活部署**：支持Docker和Kubernetes部署

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Java 17, Spring Boot 3.2, Spring Cloud | 调度中心、API网关 |
| 前端 | Vue 3, Element Plus, TypeScript, Vite | Web控制台 |
| 计算 | SecretFlow (SPU/MPC) | 计算内核 |
| 编排 | Kuscia (K8s) | 任务编排、节点管理 |
| 数据库 | MySQL 8 | 元数据存储 |
| 通信 | gRPC + TLS | 节点间通信 |

## 二、架构设计

### 2.1 系统架构

```
┌─────────────────────────────────────────┐
│              Frontend (3000)             │
│         Vue 3 + Element Plus            │
└─────────────────┬───────────────────────┘
                  │ /api
┌─────────────────▼───────────────────────┐
│           Gateway (8090)                │
│         Spring Cloud Gateway            │
└──────────┬──────────────────┬───────────┘
           │                  │
┌──────────▼──────────┐ ┌──────▼──────┐
│  Scheduler (18081)  │ │ NodeManager │
│   任务调度核心      │ │   (8082)   │
│   - TaskScheduler   │ │  - 节点注册 │
│   - Kuscia集成      │ │  - 心跳    │
└──────────┬──────────┘ └──────┬──────┘
           │                  │
┌──────────▼──────────┐ ┌──────▼──────┐
│   Kuscia Master     │ │    MySQL    │
│  (8080, 8081)      │ │   (3306)    │
│   任务编排引擎      │ │  元数据存储  │
└────────────────────┘ └─────────────┘
```

### 2.2 模块划分

| 模块 | 端口 | 职责 |
|------|------|------|
| frontend | 3000 | 前端Vue3应用 |
| gateway | 8090 | API网关，路由转发 |
| scheduler | 18081 | 任务调度核心，Kuscia集成 |
| node-manager | 8082 | 节点注册、心跳、状态管理 |
| mysql | 3306 | 元数据存储 |
| kuscia-master | 8080/8081 | Kuscia编排引擎 |

```bash

# 构建镜像
```
