# 密算平台 - 快速开始指南

## 5分钟快速部署 (Docker)

### 前提条件

- Docker 20.10+
- Docker Compose 2.0+
- 8GB+ 可用内存

### 步骤1：下载项目

```bash
git clone https://github.com/msp/msp.git
cd msp
```

### 步骤2：启动服务

```bash
cd msp-deploy/docker

# 启动所有服务
docker-compose up -d

# 等待服务启动（约1分钟）
sleep 60
```

### 步骤3：验证服务

```bash
# 检查服务状态
docker-compose ps

# 应该看到所有服务都是 "Up" 状态
```

### 步骤4：访问控制台

打开浏览器访问：**http://localhost:3000**

默认账号：
- 用户名：`admin`
- 密码：`admin123`

## 快速操作

### 创建第一个PSI任务

1. 登录控制台
2. 点击"任务管理" → "创建任务"
3. 选择任务类型：**PSI**
4. 选择参与节点：选择2个已注册的节点
5. 配置数据源：选择要匹配的数据表和列
6. 点击"提交"

### 查看任务状态

1. 在任务列表中找到您的任务
2. 点击"查看"按钮
3. 实时查看任务进度和状态

### 管理节点

1. 点击"节点管理"
2. 查看所有注册节点的状态
3. 点击节点查看详细信息

## 常用命令

### 服务管理

```bash
# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart scheduler

# 查看日志
docker-compose logs -f scheduler
```

### 节点操作

```bash
# 进入节点容器
docker exec -it msp-node-1 /bin/bash

# 查看节点状态
docker exec -it msp-node-1 python -c "import secretflow as sf; sf.reveal(sf.system_info())"
```

### 数据库操作

```bash
# 连接数据库
docker exec -it msp-postgres psql -U msp -d msp_db

# 查看任务表
msp_db=# SELECT * FROM msp_tasks;

# 查看节点表
msp_db=# SELECT * FROM msp_nodes;
```

## 常见问题

### Q: 服务启动失败？

```bash
# 检查Docker状态
docker info

# 清理后重试
docker-compose down
docker system prune -f
docker-compose up -d
```

### Q: 无法访问控制台？

```bash
# 检查端口占用
netstat -tlnp | grep 3000

# 检查前端日志
docker-compose logs frontend
```

### Q: 任务执行失败？

1. 检查节点是否在线
2. 检查数据源配置是否正确
3. 查看详细日志：`docker-compose logs scheduler`
