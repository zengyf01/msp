# 密算平台 - 快速开始指南

## 5分钟快速部署 (Docker)

### 前提条件

- Docker 20.10+
- Docker Compose 2.0+
- 8GB+ 可用内存

### 步骤1：下载项目

```bash
git clone https://github.com/msp/msp.git
cd msp
```

### 步骤2：启动服务

```bash
cd msp-deploy/docker

# 启动所有服务（Ray模式）
docker-compose up -d

# 或启动Kuscia相关服务（可选）
docker-compose --profile kuscia up -d

# 等待服务启动（约1分钟）
sleep 60
```

### 步骤3：验证服务

```bash
# 检查容器状态
docker ps --format "table {{.Names}}\t{{.Status}}"

# 应该看到以下容器都是 "Up" 状态
# msp-mysql, msp-scheduler, msp-gateway, msp-node-manager, msp-frontend
# node-a, node-b, node-c
# node-a-db, node-b-db, node-c-db

# (可选) Kuscia模式下会额外启动:
# kuscia-master, node-a-agent, node-b-agent
```

> **节点模式**：每个任务的节点模式由请求参数`nodeMode`决定（`ray`或`kuscia`）。

### 步骤4：访问控制台

打开浏览器访问：**http://localhost:3000**

默认账号：
- 用户名：`admin`
- 密码：`admin123`

## 快速操作

### 创建第一个PSI任务

1. 登录控制台
2. 点击"任务管理" → "创建任务"
3. 选择任务类型：**PSI**
4. 选择参与节点：选择2个已注册的节点
5. 配置数据源：选择要匹配的数据表和列
6. 点击"提交"

### 查看任务状态

1. 在任务列表中找到您的任务
2. 点击"查看"按钮
3. 实时查看任务进度和状态

### 管理节点

1. 点击"节点管理"
2. 查看所有注册节点的状态
3. 点击节点查看详细信息

## 常用命令

### 服务管理

```bash
# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart scheduler

# 查看日志
docker-compose logs -f scheduler
```

### 节点操作

```bash
# 进入节点容器
docker exec -it node-a /bin/bash

# 查看节点SecretFlow状态
docker exec -it node-a python -c "import secretflow as sf; print(sf.reveal(sf.system_info()))"
```

### 数据库操作

```bash
# 连接MySQL数据库
docker exec -it msp-mysql mysql -u msp -pmsp123456 msp_db

# 查看任务表
msp_db=# SELECT task_id, name, status, create_time FROM msp_tasks;

# 查看节点表
msp_db=# SELECT node_id, node_name, status FROM msp_nodes;
```

## 常见问题

### Q: 服务启动失败？

```bash
# 检查Docker状态
docker info

# 清理后重试
docker-compose down
docker system prune -f
docker-compose up -d
```

### Q: kuscia-master 启动失败（证书错误）？

```bash
# 清理kuscia数据和容器，重新初始化
docker stop kuscia-master
docker rm kuscia-master
docker volume rm docker_kuscia_data
docker-compose up -d kuscia-master
```

### Q: 无法访问控制台？

```bash
# 检查端口占用
netstat -ano | findstr "3000"

# 检查前端日志
docker logs msp-frontend
```

### Q: 任务执行失败？

1. 检查节点是否在线
2. 检查数据源配置是否正确
3. 查看详细日志：`docker-compose logs scheduler`
