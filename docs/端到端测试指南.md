# MSP 端到端测试指南

本文档介绍如何使用系统内置的「模拟节点部署」功能快速搭建测试环境，模拟节点注册、创建数据源、创建任务、执行任务的完整流程。

## 1. 概述

### 1.1 模拟节点部署功能

系统内置的「模拟节点部署」功能可以在数据库中自动创建多个独立的节点数据库，每个数据库包含示例数据表，用于演示多方数据交互流程。

### 1.2 工作流程

```
┌──────────────────────────────────────────────────────────────────┐
│                    步骤1：模拟节点部署                            │
│         点击「模拟节点部署」→ 创建节点数据库和示例数据表              │
│                  在 MySQL 中创建多个节点数据库                      │
└────────────────────────────────┬─────────────────────────────────┘
                                 │
                                 ▼
┌──────────────────────────────────────────────────────────────────┐
│                    步骤2：注册节点                                │
│     前往「节点管理」→ 手动注册节点（填写节点ID、名称、端点）        │
└────────────────────────────────┬─────────────────────────────────┘
                                 │
                                 ▼
┌──────────────────────────────────────────────────────────────────┐
│                    步骤3：注册数据源                               │
│  前往「数据源管理」→ 为每个节点注册数据源（关联数据库和表）          │
└────────────────────────────────┬─────────────────────────────────┘
                                 │
                                 ▼
┌──────────────────────────────────────────────────────────────────┐
│                    步骤4：创建任务                                │

                                 │

```

## 2. 操作步骤

### 2.1 步骤1：模拟节点部署

1. 打开 http://localhost:3000
2. 登录系统（admin / admin123）
3. 进入「节点管理」
4. 点击「模拟节点部署」按钮
5. 选择节点数量（2-4个）
6. 点击「创建模拟节点」

**创建的内容：**
- 为每个节点创建独立的MySQL数据库
- 创建示例数据表并插入样本数据
- 创建数据库访问用户

**示例配置（2个节点时）：**

| 节点ID | 节点名称 | 数据库名 | 表名 | 匹配列 |
|--------|---------|----------|------|--------|
| node-a | 数据中心A | node_a_db | data_a | id_card |
| node-b | 数据中心B | node_b_db | data_b | mobile |

### 2.2 步骤2：注册节点

1. 在「节点管理」页面，点击「注册节点」
2. 填写节点信息

**节点A注册信息：**
- 节点ID：`node-a`
- 节点名称：`数据中心A`
- 端点：`grpc://node-a:50051`
- 能力：PYU, SPU

**节点B注册信息：**
- 节点ID：`node-b`
- 节点名称：`数据中心B`
- 端点：`grpc://node-b:50051`
- 能力：PYU, SPU

3. 点击「注册」

### 2.3 步骤3：注册数据源

1. 进入「数据源管理」
2. 点击「创建数据源」

**数据源A注册信息：**
- 数据源名称：`节点A数据库`
- 数据源类型：MySQL
- 主机地址：`mysql`
- 端口：3306
- 数据库名：`node_a_db`
- 用户名：`node_a`
- 密码：`node_a123`
- 节点ID：`node-a`
- 表名：`data_a`
- 匹配列：`id_card`

**数据源B注册信息：**
- 数据源名称：`节点B数据库`
- 数据源类型：MySQL
- 主机地址：`mysql`
- 端口：3306
- 数据库名：`node_b_db`
- 用户名：`node_b`
- 密码：`node_b123`
- 节点ID：`node-b`
- 表名：`data_b`
- 匹配列：`mobile`

### 2.4 步骤4：创建PSI任务

1. 进入「任务管理」
2. 点击「创建任务」
3. 填写任务信息：

**任务配置：**
- 任务名称：`客户数据匹配`
- 任务类型：PSI
- 算法：ECDH-PSI
- 参与节点：`node-a`, `node-b`

4. 配置输入数据源：
   - `node-a`：数据源A，`id_card` 列
   - `node-b`：数据源B，`mobile` 列

5. 点击「提交」

### 2.5 步骤5：查看结果

1. 在「任务管理」找到创建的任务
2. 点击「查看详情」
3. 查看任务状态和执行日志

**预期结果：**
- 任务状态：COMPLETED
- 交集结果：`110101199001011234`, `110101199001011235`, `110101199001011236`（身份证号在A表，且对应手机号在B表）

## 4. API调用示例

### 4.1 获取访问令牌

```bash
TOKEN=$(curl -s -X POST http://localhost:8090/api/v1/msp/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"admin123"}' \
  | jq -r '.data.token')
```

### 4.2 调用模拟节点创建

```bash
# 创建2个模拟节点
curl -X POST http://localhost:8090/api/v1/msp/datasources/simulate \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"nodeId":"node-a","dbName":"node_a_db","tableName":"data_a","columnName":"id_card"}'

curl -X POST http://localhost:8090/api/v1/msp/datasources/simulate \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"nodeId":"node-b","dbName":"node_b_db","tableName":"data_b","columnName":"mobile"}'
```

### 4.3 注册节点

```bash
curl -X POST http://localhost:8090/api/v1/msp/nodes/register \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "nodeId": "node-a",
    "nodeName": "数据中心A",
    "capabilities": ["PYU", "SPU"],
    "endpoint": "grpc://node-a:50051"
  }'
```

### 4.4 注册数据源

```bash
curl -X POST http://localhost:8090/api/v1/msp/datasources \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "节点A数据库",
    "type": "MYSQL",
    "host": "mysql",
    "port": 3306,
    "database": "node_a_db",
    "username": "node_a",
    "password": "node_a123",
    "nodeId": "node-a",
    "tableName": "data_a",
    "columns": ["id_card"]
  }'
```

### 4.5 创建任务

```bash
curl -X POST http://localhost:8090/api/v1/msp/tasks \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "客户数据PSI",
    "type": "PSI",
    "algorithm": "ecdh",
    "participants": ["node-a", "node-b"],
    "inputs": {
      "node-a": {
        "column": "id_card"
      },
      "node-b": {
        "column": "mobile"
      }
    }
  }'
```

## 5. 调试和问题排查

### 5.1 查看服务状态

```bash
docker-compose ps
docker-compose logs -f scheduler
docker-compose logs -f node-manager
```

### 5.2 查看数据库中的数据

```bash
# 连接MySQL
docker exec -it docker-mysql-1 mysql -u root -proot123456

# 查看节点数据库
SHOW DATABASES;
USE node_a_db;
SHOW TABLES;
SELECT * FROM data_a;
```

### 5.3 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 模拟节点创建失败 | MySQL连接失败 | 检查mysql容器状态 |
| 节点注册失败 | 节点ID已存在 | 使用不同的节点ID |
| 数据源连接失败 | 数据库凭据错误 | 检查用户名密码 |
| 任务一直PENDING | 节点不在线 | 先注册节点再创建任务 |
